## 1. 배경 / 문제 정의

`/q/review` 의 시그니처 섹션 "지금 복습할 것"([src/app/(student)/q/review/page.tsx:174-203](../../src/app/(student)/q/review/page.tsx#L174-L203)) 은 현재:

- `unifiedQueue(cards, memoryItems).slice(0, 8)` — **상위 8개만 자르는 하드 컷** ([src/app/(student)/q/review/page.tsx:61](../../src/app/(student)/q/review/page.tsx#L61))
- 평면 `<ol>` 한 줄짜리 리스트, 각 행은 `오답/기억` 칩 + 1줄 요약 + 시간 + "다시 풀기/기억 학습" CTA
- **페이지네이션·필터·정렬·검색·overflow 표시 없음** — 9번째 이후 항목이 존재한다는 사실 자체가 UI에 드러나지 않음
- 모든 항목이 동일 무게로 노출 → 항목 수가 늘면 가독성이 선형으로 무너짐
- 모바일 좁은 폭에서 행 내부가 `오답뱃지 + 본문 + 시간 + CTA` 4-grow로 이미 압박, 항목 늘어나면 스크롤 길이도 무한 증가

**유저 지적 (2026-05-13)** — "10,000개 복습할 거리가 생기면? 페이지네이션이 적어도 고려돼야 하고, 지금 UI는 페이지네이션 자체에 적합하지도 않다. 인프라는 항상 최악 가정. 사용자는 멍청이라 쉽지 않으면 안 쓴다."

데이터 측면 현실: Leitner cards + 기억 items 모두 시간이 지나면 **수십~수백 개**가 평상값, 시험 직전 폭주 시 **수천 개**가 합당한 상한. 이 plan은 N ∈ [0, 10,000] 전체 구간에서 동작 가능한 레이아웃을 새로 설계한다.

## 2. 확장성 가정 (반드시 통과)

| 항목 | 가정값 | 설계 함의 |
|---|---|---|
| 큐 항목 수 N | 최대 10,000 | 무한 평면 리스트 금지, 페이지네이션 또는 가상화 필수 |
| 동시 보여줄 1차 가시 행 | ≤ 6~8 | 첫 화면에서 학생이 "지금 뭐부터?"를 1초 안에 결정 |
| 1행 평균 무게 | 모바일 64px / 데스크탑 56px | 가상화 시 row height 고정 가능 |
| 빈도 시나리오 | 0 / 1~3 / 4~20 / 21~200 / 200+ | 각 구간에 맞는 UI 분기 필요 |
| 네트워크 | mock 동기 (현재) → 추후 서버 페치 가능 | 페이지네이션 API 시그니처를 미리 가정 |
| 모바일 폭 | 360px | 행 내부 요소 ≤ 3 grow, CTA는 우측 1개만 |

## 3. 원칙

1. **한 큐 = 한 결정점.** 학생이 한 번에 보는 항목은 ≤ 6개 (모바일 ≤ 5). 나머지는 명시적으로 "더 있어요" 신호.
2. **"지금 해야 할 것"은 상위 N개로 한정하고, 그 외는 사용자가 의도적으로 들어가도록.** "지금 복습할 것" 자체는 큐레이션 영역, 전체 큐 탐색은 별도 화면.
3. **필터·정렬은 UI에 노출하되 기본값이 99% 케이스를 커버.** 학생은 멍청이 가정 → 기본값 그대로 풀어도 정답인 상태가 디폴트.
4. **모바일 우선.** 360px에서 행이 깨지지 않고, 페이지네이션 컨트롤도 엄지로 닿는 위치.
5. **N=0/1~3/200+ 빈 또는 폭주 시나리오를 각각 다른 메시지·CTA로 처리.** 같은 빈 상자/같은 무한 리스트 절대 금지.

## 4. 새 레이아웃 — "지금 우선 6" + "전체 큐 보기"

### 4.1 섹션 골격 (review 홈 안)

```
┌────────────────────────────────────────────────────────────┐
│ SectionHeading                                              │
│  지금 복습할 것  · 시간 지남 N / 24h 안 M                    │
│  [필터 칩 · 기본 "전체"] [정렬 ▼ "급한 순"]   [정복 세트 풀기]│
├────────────────────────────────────────────────────────────┤
│ ① ② ③ ④ ⑤ ⑥  ← 항상 6개. 모바일 5개.                         │
│   (현재 단일 행 카드 디자인 유지)                              │
├────────────────────────────────────────────────────────────┤
│ [전체 큐 보기 (총 K개) →]   [건너뛴 항목 보기]                 │
└────────────────────────────────────────────────────────────┘
```

요점:
- **노출은 6개 고정**, 9번째 이후가 존재함을 "전체 큐 보기 (총 K개)" 링크로 명시
- 필터 칩: `전체 / 오답 / 기억 / 시간 지남`  — 4개 토글, 기본 "전체"
- 정렬: `급한 순 (기본) / 잊을 위험 큰 순 / 과목별 / 최근 추가순`  — dropdown
- 빈 상태(K=0)는 별도 일러스트 + "새 문제 풀러 가기" CTA

### 4.2 별도 화면 — `/q/review/queue` (전체 큐 탐색)

홈에서 못 다 보여주는 N개를 **별도 라우트**에서 페이징·검색·필터로 탐색.

레이아웃:

```
PageHeader: "전체 복습 큐 — 총 K개"
[검색 인풋: 단원/과목/요약/SKU]    [필터 칩 동일 4개]  [정렬 동일]
[총 K개 중 X–Y 표시 · 페이지 P/PP]

(가상화 리스트 또는 페이지네이션 리스트, 한 페이지 = 20개)

[이전] [1] [2] ... [10] [다음]      ← 페이지네이션
또는
[더 보기 (다음 20개)]              ← 무한 스크롤 + 가상화 (선택)
```

페이지네이션 vs 가상화 선택 기준:
- **기본은 페이지네이션 20개/page** — 학생이 "내가 지금 몇 번째까지 봤다"를 인지하기 쉬움
- N > 500 시 자동으로 `react-virtual` 등 가상화 모드 전환 (스크롤 위치 보존)
- mock 환경에서는 client-side slice; 서버 연동 시 `?page=&pageSize=&filter=&sort=` 시그니처 일치 가정

### 4.3 행(row) 디자인 — 모바일 강화

현재 행은 데스크탑 가정으로 4 grow(`뱃지/본문/시간/CTA`). 모바일 360px에서는:

- 1차행: `[번호] [본문 1줄 truncate]  [시간]`  ← 64px
- 2차행: `[유형 칩] [BOX/과목] [원인 칩]      [CTA 점선 화살표]`
- 탭 영역 전체가 링크 (현재 동일)

데스크탑은 현재처럼 단일 행 유지, 모바일만 2-line으로 자연 분기.

### 4.4 시나리오별 분기

| K (전체 큐) | 홈 섹션 표시 | 추가 UI |
|---|---|---|
| 0 | "오늘 복습 없음 — 새 문제 풀러 가요" 일러스트 | `/q/infinity` CTA |
| 1~3 | 1~3개만 표시, "이게 다예요" 라벨 | 더보기 링크 숨김 |
| 4~6 | 그대로 표시 | 더보기 링크 숨김 |
| 7~50 | 상위 6개 + "전체 큐 보기 (총 K)" | 페이지네이션 화면 활성 |
| 51~500 | 상위 6개 + "전체 K개 — 페이지로 보기" + 시간 지남 N개 강조 배너 | 페이지네이션 화면 + 가상화 자동 발동 임계 표시 |
| 500+ | 상위 6개 + "시간 지난 N개부터 빨리 줄여요" 강한 안내 + "정복 세트 자동 추천" CTA | 폭주 가드 — 정복 세트 + 일괄 처리 안내 |

## 6. 기본값·UX 디테일

- **기본 정렬**: `급한 순` (= `hours` 오름차순; overdue가 위, "지금"이 다음)
- **기본 필터**: `전체` — 학생이 토글하지 않아도 그대로 "지금 6개" 노출
- **필터 토글 시 결과 0인 경우**: 다이얼 분기 (예: "기억 필터 — 24시간 안 항목 0개. 시간 지난 기억 N개는 따로 봐주세요.")
- **시간 지난 N개 강한 배너**: 홈 KpiBand 우측 시간 지남 카운터 클릭 시 → 전체 큐로 이동 + 필터 `시간 지남` 자동 적용
- **건너뛴 항목**: 학생이 큐에서 "나중에" 누른 항목은 별도 라벨로 표시(향후), 이 plan에서는 라우트만 예약(`?skipped=1`)
- **검색 인풋(전체 큐)**: 디바운스 200ms, 빈 검색어 시 전체. 한글/영문/숫자(SKU) 모두 매치.

## 7. 접근성·키보드

- 필터 칩: `role=tablist` 아닌 `role=group` + 토글 버튼 4개 (단일 선택)
- 정렬 dropdown: Headless 라이브러리 의존 없이 native `<select>` 우선 검토 — 모바일 OS 친화
- 페이지네이션: 좌우 화살표 키 매핑 (포커스가 paginator일 때만)
- 시간 지남 강조 배너: `role=alert` 아님(중요도 낮음) / `aria-live="polite"` 만 적용
