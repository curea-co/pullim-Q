# 2026-05-06 한자어 UX 정책 — UI 카피 vs 학습 콘텐츠 분리 원칙

> **유형**: UX 리서치 / 카피 정책
> **결정**: ✅ UI 카피는 한자어 최소화, 학습 콘텐츠는 한자어 유지
> **주 사용자**: 대한민국 고등학생
> **트리거 질문**: "고등학생이 '잔존', '임박' 같은 한자어를 이해하고 있을까? 실질적 문맹과 연관 있어 보이는데."

## 1. 배경 — 한국형 실질적 문맹의 핵심 병목

### 1.1 데이터로 본 현황

- **OECD PIAAC·PISA 한국 결과**: 단순 문해력(글자 읽기)은 최상위권이지만, 추론·어휘 기반 독해력은 중하위권. 만 16~24세 구간에서 OECD 평균 이하가 꾸준히 관찰됨.
- **EBS 다큐 「당신의 문해력」 (2021)** · **국립국어원 「국민의 언어 의식 조사」**: 고등학생 상당수가 "사흘=4일", "금일=금요일", "심심한 사과=지루한 사과"로 오독.
- "잔존(殘存)", "임박(臨迫)", "고지(告知)", "차치(且置)" 같은 한자어 의미 추정 정답률이 학년이 올라갈수록 정체.

### 1.2 구조적 원인

1. **한자 교육 사실상 폐지** — 1970년대 이후 한자 병기 축소, 2000년대 이후 한글 전용. 한자어를 **소리(음)로만** 외우고 의미 단위(형태소) 분석 능력이 안 만들어짐.
2. **한국어 어휘의 60~70%가 한자어** — 학술·법률·의료 텍스트는 90%+. 한자 형태소 감각 없이는 모르는 단어 만났을 때 추론할 발판이 없음.
3. **숏폼·이미지 중심 미디어** — 긴 문장, 한자어 밀도 높은 텍스트 노출 빈도 급감.
4. **교과서 어휘 평탄화** — 어려운 한자어가 교과서에서 빠지면서 노출 자체가 줄어드는 악순환.

### 1.3 실질적 문맹과의 연결

실질적 문맹(functional illiteracy)은 "글자는 읽지만 의미·맥락·논증을 재구성하지 못하는" 상태. **한국에서는 그 핵심 병목이 '한자어 어휘 추론 실패'에 좁혀지는 것이 특징** (영어권의 라틴/그리스 어원 추론 결손과 대응되는 자리).

즉, "잔존/임박을 모른다"는 단어 하나의 문제가 아니라, **신문 사설·계약서·약 설명서·정부 공문**을 읽을 때 모르는 단어를 추론하지 못하고 통째로 건너뛰는 행동 패턴으로 연결됨.

## 2. 풀림 서비스에 적용 — UI vs 콘텐츠 분리

풀림의 주 사용자는 **대한민국 고등학생**. 위 진단을 그대로 UX 정책에 적용하면 단순화 일변도("쉬운 말로 다 바꾸자")가 답이지만, **풀림이 교육 서비스**라는 점에서 그 결론은 절반만 맞음.

### 2.1 핵심 트레이드오프

| 영역 | 한자어 정책 | 이유 |
|---|---|---|
| **UI 카피** (메뉴·버튼·빈 상태·에러·온보딩·툴팁·알림) | **최소화** ✅ | 사용자가 헤매면 안 되는 영역. 한자어 진입 장벽 = 이탈 사유. 시험 환경이 아니므로 어휘 노출의 교육 가치 없음. |
| **학습 콘텐츠** (문항·해설·개념 설명·교재) | **유지** ✅ | 교과서·수능·내신이 한자어로 쓰여 있음. 쉬운 말로 바꾸면 시험 환경과 갭이 생기고 어휘 추론 훈련 기회를 뺏음. |

**경계선을 흐리면** "쉬워서 좋다"가 아니라 **"공부에 도움 안 되는 앱"**이 됨.

### 2.2 UI 카피 교체 가이드라인

원칙: **고등학생이 한 번 읽고 의미를 잡지 못할 한자어는 순한글 또는 일상어로 교체.**

대표 교체 후보:

| 한자어 | 교체안 | 적용 영역 |
|---|---|---|
| 잔여 학습량 | 남은 공부 / 남은 학습 | 진도·대시보드 |
| 임박한 일정 | 곧 다가오는 일정 / 마감 임박 → "마감 곧" | 알림·플래너 |
| 차시 | 회차 / N회차 | 강의·콘텐츠 카드 |
| 미달 / 미충족 | 부족 / 아직 모자람 | 점수·달성률 |
| 차치하고 | 일단 두고 | (UI 카피에 등장 X — 콘텐츠 영역) |
| 고지 / 고지함 | 알림 / 알려드림 | 시스템 알림 |
| 종료 | 끝남 / 마침 | 상태 표시 |
| 회신 | 답장 / 답변 | 메시지 UI |
| 진척도 | 진행률 | 대시보드 |
| 상기 / 하기 | 위 / 아래 | 안내 문구 |
| 익월 / 익일 | 다음 달 / 다음 날 | 일정 |
| 본인 인증 | 내 정보 확인 / 본인 확인 | 인증 플로우 |

> 풀림이 도메인 이름을 **"플래너 / Q / 클래스봇 / 라이브러리 / 스튜디오 / 스토어"**로 잡은 것 자체가 이 철학과 일치 — 한자어 도메인명("학습관리", "문제은행" 등)을 의도적으로 피한 결정.

### 2.3 학습 콘텐츠에서의 보완 — 부가가치로 전환

콘텐츠에서 한자어를 빼지 않는 대신, **한자어 자체를 학습 자산으로 만드는 기능**을 검토 가치 있음:

- **모르는 한자어 툴팁** — 문항·해설에서 한자어 long-press 시 한자 + 음 + 뜻 표시
- **어휘 메모리** — 풀림 Q의 메모리/스터디 인덱스 도메인과 자연스럽게 결합
- **단어 형태소 분해** — "잔존(殘存)" → 殘(남을 잔) + 存(있을 존) 형태소 단위로 의미 학습
- **수능·내신 빈출 한자어 묶음** — 콘텐츠 인덱싱 차원

이는 풀림이 단순 학습관리툴을 넘어 **"실질적 문맹 해소"라는 교육적 차별점**을 가져갈 수 있는 지점.

### 다음 단계 (별도 plan)
- [ ] 학습 콘텐츠용 **한자어 툴팁 기능** 명세 검토 (풀림 Q의 메모리·스터디 인덱스와 결합)
- [ ] 온보딩에서 "풀림은 교과서 한자어 그대로 다룬다 + 모르면 풀어준다"를 가치 제안으로 노출

### 영구 원칙 (CLAUDE.md 또는 design system 반영 검토)
- UI 카피 작성 시 **"고등학생이 한 번 읽고 의미가 잡히는가"**를 기본 검수 질문으로 사용
- 콘텐츠 작업 시 한자어를 **임의로 평탄화하지 않음** (시험 환경 정합 우선)
