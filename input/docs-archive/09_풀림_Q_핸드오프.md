## 1. 서비스 개요

### 1.1 한 줄 정의
**Q는 스튜디오·스토어에서 흘러온 문항 콘텐츠를 학생이 실제로 풀고·틀리고·복습하고·분석하는 "콘텐츠 플라이휠의 소비단(端)"이다. 한 문제를 풀 때마다 플라이휠이 한 바퀴 돈다.**

### 1.2 포지셔닝
| 축 | Q의 위치 |
|---|---|
| 플라이휠 | Studio(생산) → Store(유통) → **Q(소비·분석)** → Data(피드백) → Studio |
| 주 사용처 | 문제 풀이·오답 정복·모의고사 (학교 밖) |
| 경쟁 차별점 | 문제은행이 아니라 "풀이·오답·해설·모의고사·분석을 하나의 엔진으로 통합" + 판도를 바꾸는 **풀림 해설** |

### 1.3 타겟 유저
1. **P1 (주) 학생**: 문제 풀이·오답 복습·모의고사를 한 곳에서 해결하고 싶은 중고생
2. **P2 교사**: 자신이 만든 문제의 풀이 데이터를 통해 콘텐츠를 개선하고 싶은 스튜디오 제작자
3. **P3 학부모**: 자녀의 실력 성장(θ)을 근거 있게 이해하고 싶은 부모
4. **P4 콘텐츠 제작사**: 기출·교재를 유통하고 학생 풀이 데이터로 가치를 확장하고 싶은 출판사·학원

### 1.4 핵심 가치 제안 (Value Props)
- **V1 · 통합 엔진**: 문제풀이·인덱스·오답정복·모의고사·분석이 분절되지 않은 하나의 서비스
- **V2 · 풀림 해설**: 12-섹션 "이해의 여정"으로 기존 출판사 해설을 대체 (서비스의 시그니처)
- **V3 · 콘텐츠 플라이휠**: 네가 푼 문제 데이터가 다음 문제의 품질을 올린다
- **V4 · θ 기반 성장**: 점수가 아닌 IRT θ로 실력을 측정하고 성장 곡선을 그린다
- **V5 · 크로스 과목**: 수학·영어·국어·통합사회·통합과학 한 엔진에서

## 2. 디자인 원칙

1. **해설은 답이 아니라 지도다.** 모든 해설은 다중 경로·뿌리·응용을 함께 제시한다.
2. **문제 한 번 풀이 = 플라이휠 한 바퀴.** 모든 풀이 로그는 익명화되어 Studio로 환류된다.
3. **θ는 상수가 아니라 곡선이다.** 단일 점수 UI 금지, 반드시 시간축 추세로 표시.
4. **Scope Guard는 Q에서도 유효하다.** AI 코치는 ClassBot과 동일한 5단계 권한 모델을 따른다.
5. **오답은 자원이다.** 삭제·숨기기 불가, 5단계 Leitner 박스로 반드시 재회.

## 4. 주요 기능 · 상세

### 4.1 홈 · 플라이휠 (Home · Flywheel)

**핵심 시각 요소**:
- 플라이휠 다이어그램 SVG (Studio → Store → Q → Data 4 노드, 화살표 순환)
- 오늘의 풀이 큐 (5문항, 인터리빙된 소스: 스튜디오/기출/오답)
- 단원별 숙련도 히트맵 (12주 × 5단원)
- 플라이휠 유입 피드 (스튜디오·스토어에서 오늘 들어온 콘텐츠)

**KPI 카드 5개**:
- 오늘 풀이 (23/40)
- 정답률 (78.3%)
- 오답 대기 (23, overdue 2)
- θ (1.32)
- D-342 모의 점수 (84/100)

### 4.2 문제 풀이 (Solve) — 3-pane 워크스페이스

**레이아웃**:
- **좌(문제 영역)**: paper-texture 문제 카드 · SKU ID · 출처 메타데이터 · 발문 · 도형 SVG · 5지 선다
- **중앙(풀이 노트)**: 노트 배경 · 손글씨/LaTeX 전환 · 자동 저장 · 풀이 tools (펜/수식/지우개/STT)
- **우(AI 코치 사이드)**:
  - AI 풀이 코치 (Scope Guard L3 기본)
  - 미니 KPI 4개 (남은 시간·예상 θ·힌트 사용·유사 문제)
  - 연관 오답 링크

**핵심 인터랙션**:
- Scope Guard L1-L5 학생이 선택 (이력 기록)
- AI 코치에 "더 물어보기" 버튼 (⌘+/)
- 힌트 사용 시 해당 문제 점수 가중치 감소 (공정성)
- 풀이 노트 자동 저장 (`blur` 및 매 30초)

### 4.3 풀림 해설 (Explain) — 시그니처 기능 ★

**포지션**: 다른 출판사 해설을 대체하는 Q의 랜드마크 기능.

**과목 탭 (5과목)**: 수학 · 영어 독해 · 국어 비문학 · 통합사회 · 통합과학
(각 과목별 샘플 해설 1개 이상 필수)

**각 해설의 12-섹션 구조** (순서 고정):

| # | 섹션 | 내용 |
|---|---|---|
| 1 | **Hero Recap** | 문제 요약 + 네 답 vs 정답 나란히 |
| 2 | **Prologue · 왜 이 문제인가** | 교육과정상 이 문제의 존재 이유 |
| 3 | **4-Path Solution Spine** ★ | 같은 답, 4개 독립 경로 (정석/기하직관/좌표/심화) — 시그니처 컴포넌트 |
| 4 | **Textbook Root Graph** | 중·초→중→고 교과 DAG + 앞으로 쓰일 곳 |
| 5 | **Error Anatomy** | 네 풀이 노트 vs 정답 노트 · 실수 지점 annotation |
| 6 | **100명의 선택** | 5지 선다 분포 + 각 선지를 고른 심리학적 이유 |
| 7 | **Visual Intuition Canvas** | 인터랙티브 SVG로 핵심 통찰 1개 |
| 8 | **Pattern Family** | 같은 패턴의 8개 친척 문제 (수능·모평·스튜디오·AI 예측) |
| 9 | **Feynman Challenge** | 마이크 UI · 2분 안에 설명 · AI가 STT + 체크리스트 평가 |
| 10 | **Teacher Voices** | 같은 해설 3가지 톤 (정석·친구·스파르타) |
| 11 | **History + Real-World** | 개념의 역사 + 현실 응용 |
| 12 | **Memory Anchor + Next Review** | 암기 닻 카드 + Leitner 복습 스케줄 |

**4-Path Solution Spine · 구현 세부**:
- 4개 컬럼, 각 컬럼 독립 경로
- 각 경로에 이름·태그·시간·우아함 평점·일반화 평점
- 단계 카드 (step) + 교과서 인용 (cite, e.g. "→ 수학I §2.3.1 p.88")
- 한 경로에 "RECOMMENDED" 뱃지 (황금 링)
- 각 경로 하단: 정답률 추정 (해당 경로 사용자 기준)

### 4.4 단원 인덱스 (Index)

**기능**: 교과의 모든 단원·절·개념을 트리로 보여주고, 학생 숙련도를 색으로 표시.

**구성**:
- 과목 필터 (수학I, 영어 독해 등)
- 정렬 필터 (교과 순서 / 약점 우선 / 최근 풀이 / 출제 빈도)
- 트리: L1 (대단원) → L2 (중단원) → L3 (개념)
- 각 노드에 숙련도 셀 (5단계 색상 + weak 빨강)
- 약점 단원은 자동 하이라이트, "오답 정복으로 보내기" CTA

### 4.5 오답 정복 (Conqueror) — Leitner 5-Box

**시스템**:
- 5개 박스, 복습 간격: BOX 1 (1일) → 2 (3일) → 3 (7일) → 4 (14일) → 5 (30일)
- 연속 성공 시 다음 박스, 실패 시 BOX 1 복귀
- BOX 5에 5회 연속 성공 시 "마스터" 상태 (30일 주기 유지)

**UI**:
- 다크 테마 (집중력 상승)
- 각 오답 카드: 문제 ID · 발문 요약 · 에러 패턴 · BOX 번호 · 다음 복습일
- Overdue 표시 (빨간 좌측 바)
- 필터: 모두 / 오늘 복습 / Overdue / 단원 그룹 / 에러 패턴

### 4.6 모의고사 (Mock Exam)

**구성**:
- 시험 세트 선택 (기출·스튜디오·AI 생성)
- 종이 + OMR 2-pane 화면
- 상단 hero 타이머 (큰 숫자)
- OMR 버블: 클릭 = 마킹 / Shift-클릭 = 표식
- AI 감독관: 탭 이탈·부정 의심 기록
- 제출 후 즉시 분석으로 이동

**시험 종류**:
- 수능 기출 복원 (연도별)
- 모평·학평
- 스튜디오 수제 (선생님이 만든 세트)
- AI 생성 (예측 문항, 난이도 자동 보정)

### 4.7 풀이 분석 (Analysis)

**4개 메인 차트**:
1. **θ 진행 라인** — 단원별 12주 추이 + 목표선 (θ=1.5)
2. **문항당 풀이 시간 분포** — 최근 40문항 바 차트 + 목표 시간 기준선
3. **에러 패턴 Top 8** — 공식 대입 오류·논지 전환 누락 등 태그
4. **시간대별 정답률** — 7시~23시 라인 + 피크 하이라이트

**액션 카드** (레몬 배경):
- 분석 결과 기반 자동 제안 3개
- 플래너 블록 자동 배정 연동

### 4.8 내 라이브러리 (Library)

**구성**:
- 풀었던 문제 · 북마크 · 하이라이트 · 내 노트 종합
- 필터: 전체 / 북마크 / 오답 / 마스터 / 내 노트
- 카드 그리드 (자동 맞춤 열)
- 태그 검색 + SKU 검색

### 4.9 스튜디오 연동 (Studio Link)

**구조**: 학생이 구독한 Studio 채널의 콘텐츠 팩을 Q 라이브러리로 흡수.

**카드 구성**:
- SKU 코드 (STD-KMATH-24-42)
- 팩 타이틀 · 제작자 + VERIFIED 뱃지
- 설명 · 문항 수 · 평점
- 가격 · "구독 중" 상태 표시
- "큐 추가" 버튼

**플라이휠 환류 UI**:
- 다크 패널 "YOUR DATA → BETTER CONTENT"
- 학생 풀이 로그가 Studio로 흐름을 명시

### 4.10 스토어 (Store)

**구조**: 외부 콘텐츠 마켓플레이스, 구매 시 Q 라이브러리에 자동 추가.

**카테고리**:
- 기출 (수능·모평·학평)
- 교재 (메가·종로·EBS)
- 인강
- 스튜디오 (개인 크리에이터)
- 무료

**상업 모델**: 구매 → 라이센스 → Q에서 풀기 가능. 재판매·공유 불가.

### 4.11 시작하기 · 도움말

- 5단계 온보딩 (학년 설정 → 진단 10문항 → 약점 확인 → 콘텐츠 연결 → 첫 세션)
- FAQ: θ 개념, Scope Guard, Leitner, AI 감독관 등

## 5. 핵심 사용자 플로우

### Flow A · 신규 학생 첫 풀이 사이클
```
1. 가입 → 학년·목표 입력
2. 진단 10문항 자동 출제 (난이도 적응)
3. θ 추정값 계산 → 홈 대시보드
4. 단원 히트맵 확인 → 약점 단원 발견
5. "문제 풀이" 탭 → 첫 블록 5문항
6. 오답 1개 → 오답 정복 BOX 1 자동 배정
7. 풀림 해설 12-섹션 관람 (6-10분)
8. 내일 BOX 1 복습 일정 생성
```

### Flow B · 일반 학생의 하루 루틴
```
1. Planner NEXT BLOCK "수학 사인법칙 8문제"
2. Planner → Q Solve 자동 진입 (딥링크)
3. 8문항 풀이 (평균 40분)
4. 오답 2개 발생 → 자동 BOX 배정
5. 풀림 해설 (오답만 선택 열람)
6. 완료 → Planner로 복귀 + XP 부여
7. 저녁: 오답 정복 탭 → BOX 1 2개 재도전
```

### Flow C · 모의고사 풀 사이클
```
1. 모의고사 탭 → "2025 6월 모평 수학" 선택
2. 실전 100분 타이머 시작
3. OMR 마킹 + 표식 활용
4. 제출 → 자동 채점 + θ 계산
5. 풀이 분석 이동 → 틀린 문제 클러스터링
6. 각 오답 → 풀림 해설 열람
7. 오답 정복에 일괄 배정
8. 플래너에 "2주 취약 보강" 자동 블록 생성
```

### Flow D · 풀림 해설 깊이 탐색
```
1. Solve에서 오답 문제 클릭 → 해설 열람
2. Section 1 Hero → 내 답 ② vs 정답 ④ 비교
3. Section 3 4-Path → RECOMMENDED 경로 집중
4. Section 5 Error Anatomy → 내 실수 지점 확인
5. Section 7 Visual Canvas → 핵심 통찰 애니메이션
6. Section 9 Feynman Challenge → 2분 마이크 녹음
7. AI STT 분석 → 체크리스트 결과 피드백
8. Section 12 Memory Anchor → 카드 저장
9. 오답정복 복습 주기 자동 설정
```

### Flow E · 학생 데이터 → Studio 환류
```
1. 100명 학생이 같은 문제 풀이
2. 평균 풀이 시간·정답률·에러 패턴 익명 집계
3. Studio 제작자 대시보드에 "이 문제 난이도 θ 1.4 → 1.6 조정 제안"
4. 제작자가 해설 보강 (Section 5 Error Anatomy 추가)
5. 다음 학생이 풀 때 개선된 해설 수신
6. 플라이휠 완성
```

## 6. 유스케이스

### UC-1 · 고3 수험생의 오답 정복
- 페르소나: 김나윤 (고2 → 고3)
- 문제: 오답 노트를 수기로 적어도 다시 안 봄
- 해결: Leitner 5-box 자동 스케줄 + 풀림 해설 깊이 탐색
- 성과: 오답 재도전 성공률 38%→64%, θ +0.3/3개월

### UC-2 · 재수생의 모의고사 루틴
- 페르소나: 박민지 (재수·독학)
- 문제: 모의고사 본 후 해설·분석이 피상적
- 해결: 모의고사 → 즉시 분석 → 풀림 해설 → 자동 취약 보강

## 7. 핵심 개념·시스템

### 7.1 문제 SKU 체계

```
Q-<SUBJECT>-<UNIT>-<NNNN>
```

- `Q-MATH-I-0239` · 수학I 239번
- `Q-ENG-RDG-1205` · 영어 독해 1205번
- `Q-KOR-NSL-0871` · 국어 비문학 871번
- `Q-SOC-0542` · 통합사회 542번
- `Q-SCI-0231` · 통합과학 231번

**메타데이터**:
- 출처 (기출/스튜디오/스토어/AI)
- 난이도 θ
- 태그 (교육학 엔진·단원·유형)
- Scope Guard 허용 기본값

### 7.2 θ (Theta) · IRT 기반 숙련도

- 범위: 0.0 - 2.4
- 문항별 θ: 100명의 평균 추정 필요 시점 난이도
- 학생별 θ: 풀이 로그 기반 IRT 추정 (3-parameter logistic)
- 단위: 주기적 재계산 (주간)
- 1.6+ = 수능 1등급 수준 추정

### 7.3 Leitner 5-Box Spaced Repetition

| BOX | 복습 주기 | 의미 |
|---|---|---|
| 1 | 1일 | 최근 실패, 긴급 |
| 2 | 3일 | 1회 성공, 단기 강화 |
| 3 | 7일 | 2회 성공, 중기 저장 |
| 4 | 14일 | 3회 성공, 장기 저장 |
| 5 | 30일 | 4+회 성공, 유지만 |

**규칙**: 실패 시 BOX 1 복귀, 성공 시 다음 BOX.

### 7.4 에러 패턴 (Error Pattern)

AI가 오답을 분석해 패턴으로 분류.

**대표 유형**:
- 공식 대입 오류 (수학)
- 논지 전환 누락 (영어)
- 근거 위치 판단 (국어)
- 경계 조건 누락 (수학)
- 단일 자료 해석 (통합사회)
- 평형 vs 속도 혼동 (통합과학)
- 반전어 오독 (비문학)
- 어휘 의미 혼동 (영어)

패턴은 자동으로 플래너 취약 보강 블록으로 전환.

### 7.5 풀림 해설 품질 기준

**10개 체크리스트** (Studio 제작자용):
1. Hero에 사용자 답 vs 정답 표시 가능
2. Prologue에 "이 문제의 교육적 존재 이유" 서술
3. 4-Path 중 최소 3-Path는 독립 접근
4. 각 단계에 교과서 인용 (최소 1개/경로)
5. Root Graph에 전·후 연결 각 4개 이상
6. Error Anatomy에 대표 오답 2개 서술
7. 100명의 선택에 각 선지별 심리 근거 서술
8. Visual Canvas는 인터랙티브 (재생 버튼 포함)
9. Pattern Family 최소 5개
10. Memory Anchor는 한 줄 암기문 포함

**AI 지원**: Studio 제작 시 각 섹션 템플릿 제공, 초안 자동 생성 후 교사 검수.

### 7.6 Scope Guard (Q 버전)

ClassBot과 동일 5단계, Solve 화면에 UI 노출:
- 시험 모드 (모의고사) = 자동 L1
- 일반 풀이 = 기본 L3
- 오답 정복 복습 = 기본 L5
