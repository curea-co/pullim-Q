## 1. 전략적 포지셔닝

> **"교육자의 생산성을 10배로 만드는 AI 창작실"**

스튜디오는 풀림 생태계의 **콘텐츠 공급원**이다. 스튜디오의 성공이 스토어의 콘텐츠 풍부함을, 스토어의 콘텐츠가 스터디의 학습 경험 품질을 결정한다.

**핵심 목표**: 크리에이터가 최소 노력으로 최대 품질의 콘텐츠를 생산하게 하는 것.

## 3. 기능 상세

### 3.1 S1. AI 문제 생성

풀림의 **가장 핵심적인 기능**. 기출 문제를 기반으로 실제 시험과 동일한 구조의 유사 문제를 AI가 생성한다.

#### 4가지 생성 방식

| 방식 | 설명 |
|------|------|
| **기출 정보로 찾기** | 학년·출제 연월 선택 → 기출 탐색 → 유사 문제 생성 |
| **문제 유형으로 찾기** | 문제 유형·사고 유형 조합 필터 → 유형 기반 생성 |
| **파일 첨부하기** | PDF/이미지 업로드 → OCR 자동 변환 → 유형화 → 유사 문제 생성 |
| **텍스트 입력하기** | 문제 레이아웃 선택 → 직접 입력 → 유사 문제 생성 |

#### 지원 과목
- **1차**: 국어, 영어, 수학
- **2차**: 사회탐구, 과학탐구

#### 생성 결과물
문제 본문, 해석(한글), 해설(한글), 스크립트(듣기), 문제 유형 정보, 난이도, 주제, 생성 시간

#### 핵심 기술

| 구성 요소 | 기술 | 역할 |
|----------|------|------|
| CUREA DEEP 엔진 | LLM Fine-tuning + Prompt Engineering | 교육과정 3-Depth 분류 기반 구조적 문제 생성 |
| 하이브리드 RAG | Pinecone/Weaviate + 교과서·EBS 지식베이스 | 생성 내용의 사실 적합성 검증, 환각 최소화 |
| Self-Refine | Multi-pass AI 자가 교정 파이프라인 | 정답률 12~17%p 향상 (자체 검증 완료) |
| 멀티 모델 | Gemini Flash/Pro, Claude Opus | 용도별 최적 모델 선택 (비용·품질 밸런스) |

#### 생성 파이프라인
1. 사용자가 생성 방식과 기준을 선택·입력
2. 입력 데이터를 교육과정 분류 체계(대단원→중단원→소단원)에 매핑
3. RAG로 관련 교과서·EBS 지식을 검색하여 컨텍스트 구성
4. LLM이 구조적 프롬프트(문제 유형별 템플릿)에 따라 문제·해설·해석 생성
5. Self-Refine: 생성된 문제를 AI가 재검토 → 오류 발견 시 자동 수정 (2~3회 반복)
6. 정답 검증 로직(수식 검증, 논리 검증)으로 최종 필터링

#### 비용 추정
- LLM API 호출: 문제 1건당 약 ₩200~500
- RAG 인프라: Pinecone Serverless 월 $70~200
- GPU 서버 (파인튜닝): AWS SageMaker 월 $500~1,500

#### 경쟁사 비교

| 경쟁사 | 풀림 대비 약점 |
|--------|--------------|
| 콴다(Mathpresso) | 문제 생성 불가, 풀이 검색만 |
| 뤼이드(Riiid) | 자체 문제 생성 불가, 문제은행 의존 |
| ChatGPT/Claude | 교육과정 비특화, 분류 체계 없음, 환각 통제 불가 |
| 비상교육 | AI 생성 역량 부재, 생산 속도·비용에서 열세 |

### 3.2 S2. 문제 등록 (OCR 기반 구조화)

기존 보유 자료(PDF, 이미지)를 풀림 포맷으로 변환·등록하는 기능.

#### 처리 파이프라인
1. 업로드된 PDF/이미지를 페이지 단위로 분할
2. Vision 모델이 문항 영역을 감지하고 개별 문항으로 분리
3. 각 문항에서 발문, 질문, 보기, 선택지 구조를 인식
4. 수식이 포함된 경우 LaTeX로 변환
5. 정답 후보를 추정하고, 유형·난이도를 분류 모델로 자동 태깅
6. 교육과정 분류 체계에 자동 매핑
7. 사용자가 검토 후 수정·승인

#### 핵심 기술

| 구성 요소 | 기술 |
|----------|------|
| OCR 엔진 | Vision LLM + 자체 Fine-tuning (한국 시험지 특화) |
| 문항 분리 | Object Detection (YOLO 변형) |
| 수식 인식 | LaTeX 변환 모델 |
| 분류 모델 | Custom Classifier (3-Depth 교육과정 자동 매핑) |

### 3.3 S3~S6. 학습 자료 제작 도구

**시험지**: 생성·등록된 문항을 조합하여 시험지로 구성. 문항은 **ID 참조 방식**으로 관리되어 동일 문항을 여러 시험지에 포함 가능. 판매 자료는 스냅샷으로 보호.

**문제집**: 시험지 여러 개를 묶어 문제집으로 구성.

**개념서·워크북**: 섹션 구조의 문서형 자료. AI 초안 생성 후 에디터에서 편집. 과목별 섹션 자동 생성 템플릿 제공.

**슬라이드**: 수업용 프레젠테이션을 AI가 생성하고 캔버스에서 편집·내보내기.

**미디어**: 프리셋 기반 이미지 생성, fill-in 에셋 생성.

### 3.4 S7. 콘텐츠 검증 시스템 (Quality Gate)

풀림의 핵심 차별점을 구성하는 **이중 검증 체계**.

#### 정량 평가 (AI 자동)
- 교과 커리큘럼 부합도
- 구조적 정확성 (수식/그래프/논리)
- 환각 지수 (하이브리드 RAG 기반 지식베이스 대조)

#### 정성 평가 (전문가)
- 실전 적합성 (5단계 척도)
- 해설 친절도
- 변별력 지수

#### 등급 관리
- 4.5점 이상 → "풀림 인증 문항" 등록
- 낮은 점수 → AI 재학습 데이터로 격리

#### RLHF 피드백 루프
전문가 평가 데이터 → 모델 미세조정 → 품질 지속 향상

**연구 근거**: OpenAI InstructGPT(2022)에서 RLHF가 모델 출력 품질을 유의미하게 향상시킨 것 검증. CMU Self-Refine(2023)에서 반복적 자가 교정 효과 실증.

**경쟁사 비교**: "AI 생성→AI 자가 검증→전문가 검증→RLHF 피드백" 풀 사이클을 구현한 한국 에듀테크는 없음. 이것이 풀림의 가장 강력한 기술적 해자.

### 3.5 S8. 인터랙티브 시각화 엔진 (비주얼 스튜디오)

교육 특화 인터랙티브 시각화를 생성하고, 가이드 텍스트·탐구 질문·확인 문제·해설과 결합하여 **비주얼 학습 모듈(VLM)**로 패키징하는 기능.

#### VLM 구조
```
비주얼 학습 모듈 (VLM)
├── 인터랙티브 시각화 (메인 콘텐츠)
├── 학습 가이드 (도입 텍스트, 단계별 탐구 안내, 핵심 개념 요약)
├── 탐구 질문 (2~5개)
├── 확인 문제 (CUREA DEEP 생성)
├── 해설
└── 메타데이터 (교육과정 3-Depth, 난이도, 예상 학습 시간)
```

#### 생성 방식 3가지
- **모드 A**: 자유 프롬프트로 시각화 요청
- **모드 B**: 교육과정 기반 선택 (학년→과목→단원→세부 개념)
- **모드 C**: 기존 문제 연결 (스튜디오에서 생성한 문제 선택 → 관련 개념 VLM 자동 생성)

#### 템플릿 라이브러리 (200+ 목표)

| 과목 | 템플릿 수 | 예시 |
|------|----------|------|
| 수학 | 50+ | 함수 그래프 탐구기, 기하 작도, 확률 시뮬레이터, 미적분 시각화, 벡터 연산기 |
| 과학 | 80+ | 포물선 운동, 파동 간섭, 전기 회로, 분자 운동, 유전 교배, 주기율표 |
| 사회 | 30+ | 수요-공급, 인구 피라미드, 역사 타임라인, 선거 시뮬레이터 |
| 영어 | 20+ | 문장 구조 파서, 시제 타임라인, 어근·접두사 트리, 독해 구조도 |

#### 핵심 기술

| 구성 요소 | 기술 |
|----------|------|
| 시각화 렌더링 | React + TypeScript + D3.js + Three.js |
| 물리 엔진 | Matter.js |
| 수학 엔진 | Math.js + KaTeX |
| 애니메이션 | Framer Motion |
| 샌드박스 | iframe sandbox |
| 노코드 에디터 | Custom Visual Editor |

#### Claude Interactive Visuals 대비 차별점

| 차원 | Claude | 풀림 비주얼 스튜디오 |
|------|--------|-------------------|
| 생성 품질 | 매번 달라짐 | 템플릿 기반 안정적 품질 |
| 교육과정 연결 | 없음 | 3-Depth 자동 매핑 |
| 문제 연결 | 없음 | CUREA DEEP 연동 확인 문제 |
| 학습 데이터 | 없음 | 조작 행동 수집·분석 |
| 저장·유통 | 불가 | 스토어 판매, 교사 콘솔 가능 |
| 교사 편집 | 프롬프트 재시도만 | No-Code 비주얼 에디터 |

#### 비용 추정
- 템플릿 개발: 시니어 프론트엔드 2~3명 + 교육 전문가 5명, 6~12개월
- 템플릿 1개당 약 ₩500,000~1,500,000
- 200개 템플릿: 약 ₩1~3억
- 노코드 에디터: 프론트엔드 2명, 3~4개월
- 총 초기 투자: ₩4~6억

### 3.6 S9. AI 해설 영상 자동 생성

문제·개념에 대한 해설을 AI 아바타 강사가 3~10분 숏폼 영상으로 설명하는 콘텐츠를 자동 생성.

#### 파이프라인
1. CUREA DEEP 해설 생성 → 구어체 스크립트 변환
2. TTS (ElevenLabs/TypeCast/CLOVA Voice)
3. AI 아바타 (HeyGen/Synthesia API) + 시각 자료 오버레이
4. 자동 자막 + 문제 이미지 합성

**비용**: 영상 1건(5분) 약 ₩4,000~12,000

### 3.7 S10. AI 오디오 콘텐츠 생성

**콘텐츠 유형**: 통학길 개념 요약, 수능 영어 듣기 팩, 문학 작품 해설, 단어 암기 오디오, 수학 개념 팟캐스트

### 3.8 S11. 서술형 문제 생성 + AI 채점 엔진

2028 수능 서술형 전환 대비. 서술형·논술형 문제를 AI가 생성하고, 채점 루브릭을 자동 구성하며, 학생 답안에 대해 AI가 루브릭 기반 자동 채점 + 상세 피드백을 제공.

#### 핵심 기술

| 구성 요소 | 기술 |
|----------|------|
| 서술형 문제 생성 | CUREA DEEP 확장 모듈 (짧은 서술형, 논술형, 자료 분석형 템플릿) |
| 루브릭 생성 | LLM + 교육과정 DB (내용 정확성, 논리 구조, 표현력 등) |
| AI 채점 엔진 | NLP 분석 + Rubric Matching |
| 교사 검토 UI | React + 채점 워크플로우 |

**연구 근거**: ETS e-rater, Turnitin Feedback Studio 등에서 90% 이상의 인간-AI 일치율 달성.

**비용**: ML 엔지니어 2명, 6~9개월, 총 ₩3~5억

## 4. 스튜디오 ↔ 3축 연동

### 4.1 → 스토어 연동
- 스튜디오에서 제작한 모든 콘텐츠를 스토어에 즉시 등록·판매
- 판매 시점의 콘텐츠를 불변 스냅샷으로 보존 (원본 수정 영향 없음)
- 콘텐츠 검증 등급에 따라 "풀림 인증 마크" 부여

### 4.2 → 스터디 연동
- 적응형 엔진이 적절한 난이도의 기존 문제가 없을 때 스튜디오 엔진에 실시간 문제 생성 요청
- 스터디 학습 데이터(정답률, 풀이 시간, 변별력)가 스튜디오로 역류하여 문제 메타데이터 자동 보정
- AI 소크라테스 튜터가 CUREA DEEP의 문제 분석(출제 의도, 핵심 개념)을 활용

### 4.3 데이터 플라이휠 기여
```
스터디 학습 데이터 → CUREA DEEP 모델 개선 → 스튜디오 콘텐츠 품질↑ → 스토어 콘텐츠 풍부↑ → 더 많은 학습자 유입 → 루프 반복
```

## 5. 시장 침투 전략

- **채널**: 교사 커뮤니티(인디스쿨, 참쌤스쿨), 학원 협회, SNS 마케팅
