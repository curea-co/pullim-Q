## 0. 이 문서를 보내는 이유

승훈 씨가 현재 만들고 있는 `/q/analysis`는 **학습자 전체 단위의 분석**(자기 점검 점수, 풀이 습관, 단원별 실력, 예상 등급, 또래 비교)이 잘 잡혀 있다. "내 실력, 두 각도로 보기"라는 컨셉이 명확하고, 진단 → 강점/약점 → 다음 액션 흐름이 정리되어 있다.

별도로 진행 중인 Pullim FaaS 트랙에서 "**문제 하나를 중심으로 한 학습 허브**"라는 컨셉을 정의했다. 이건 풀림Q의 분석 페이지와 충돌하는 게 아니라, **풀림Q가 지금 비어 있는 한 층을 채워주는 보완**이 된다.

이 문서의 목적은 다음 세 가지다.

1. 현재 `/q/analysis`의 강점과 비어 있는 층을 정리한다.
2. Pullim FaaS 정의서의 5개 핵심 기능을 풀림Q UX 안에 어떻게 끼워 넣을 수 있는지 제안한다.
3. 화면 위치, 라우팅, 데이터 구조 후보까지 함께 정리해 개발 의사결정을 빠르게 만든다.

원본 정의서(FaaS 문서)를 통째로 읽지 않아도 이 문서만으로 결정 가능하도록 만든다. 단, 5번 기능 상세, 산출물, MVP 범위, 저작권 원칙은 원본을 직접 참조해야 한다.

## 1. 현재 `/q/analysis` 페이지 요약 (전달자 이해 기준)

확인한 페이지 구성:

- 상단: 풀림 Q 브랜드, "17일째" 학습 streak.
- 네비: 무한풀기 / 코치 / 분석 / 복습 / 소개하기.
- 타이틀: "풀림 분석 — 내 실력, 두 각도로 보기".
- 진단 메타: "마지막 진단 2일 전 / 단원별 실력 + 풀이 습관".
- 블록 A: **자기 점검** (54점, 또래 60점, "선지 변경이 너무 적어요" 피드백, "풀이 습관 자세히" 진입).
- 블록 B: **강점 분석** (수학, 예상 3등급, 상위 33%, "단원별 자세히" 진입).
- 액션: 풀이 습관 자세히 / AI에게 묻기 / 단원별 자세히.

판단: 이 페이지는 학생을 **"전체 학습자"로 추상화한 분석 레이어**다. 즉, 점수·습관·단원 단위의 거시 진단이 잘 잡혀 있다.

---

## 2. 지금 비어 있는 층 — "문제 한 개" 단위

`/q/analysis`에서 현재 보이지 않는 층은 다음이다.

- 학생이 **어제 푼 그 문제 하나**가 왜 틀렸는지.
- 그 문제 안에서 **놓친 단서, 함정 표현, 개념, 어휘**가 무엇인지.
- 그 문제를 **다음에 같은 유형으로 만났을 때의 풀이 전략**이 무엇인지.
- 그 문제와 연결된 **유사 문제·변형 문제**가 무엇인지.
- 그 문제가 **자동으로 오답노트와 복습 큐로 어떻게 흘러가는지**.

현재 풀림Q는 "분석" 탭에서 **습관 진단**과 **단원별 진단**을 보여주지만, 그 사이를 잇는 **"문제 단위 진단 + 학습 허브"** 층이 비어 있다. 학생 입장에서는 "내 실력이 3등급인 건 알겠는데, 어제 그 빈칸 문제를 왜 틀렸고 뭘 봐야 하는지"가 한 화면 안에서 안 열린다.

이 빈 층이 Pullim FaaS 정의서가 채우려는 부분이다.

---

## 3. 보완의 핵심 컨셉: "분석 페이지에 문제 단위 드릴다운을 붙인다"

핵심 가설:

> 풀림Q `/q/analysis`의 거시 진단(습관/단원)과, Pullim FaaS의 미시 학습 허브(문제 한 개)는 **상하 레이어 관계**다. 같은 분석 메뉴 안에서 거시 → 미시로 드릴다운되어야 학습이 점수 확인에서 끝나지 않고 다음 행동까지 이어진다.

구체적으로 풀림Q에 붙일 보완은 다음 두 갈래다.

**A. 분석 페이지 안의 신규 블록**
현재 "자기 점검 / 강점 분석" 옆 또는 아래에 **"문제 단위 진단"** 블록을 추가한다. 최근에 푼 문제 중 학습 가치가 높은 문제 3~5개를 카드로 띄우고, 카드 안에 오답 원인 한 줄 + "이 문제 풀이 다시 보기" CTA를 둔다.

**B. 신규 라우트 `/q/analysis/[questionId]` (또는 `/q/question/[id]`)**
카드 또는 무한풀기 결과창에서 진입하는 **문제 단위 학습 허브 페이지**. 이 페이지가 FaaS 정의서의 5개 핵심 기능을 전부 담는 그릇이 된다.

이 둘만 추가되면 풀림Q는 "거시 분석 → 미시 학습 허브 → 복습"으로 흐름이 닫힌다.

## 4. 추가/보완할 핵심 기능 5가지

원본 FaaS 정의서의 5개 핵심 기능을 풀림Q UX 안에서의 위치와 함께 정리한다.

### 기능 1. 오답 원인 진단 카드

**무엇:** 학생이 틀린 이유를 유형화해 보여준다. (지문 근거 놓침 / 단어 의미 오해 / 선지 범위 과대 해석 / 조건 누락 / 개념 혼동 / 계산 실수 / 자료 해석 순서 오류 / 유형 전략 모름 / 배경지식 부족 / 문장 구조 해석 실패 등)

**풀림Q에서의 위치:**
- `/q/analysis` 상단에 "최근 오답 원인 Top 3" 미니 카드 1줄.
- `/q/analysis/[questionId]` 최상단 hero 카드로 확장 표시.

**현재 페이지와의 관계:** 자기 점검의 "선지 변경이 너무 적어요" 같은 습관 피드백과 **다른 차원**이다. 이건 문제 한 개 단위의 인지적 원인이다. 습관 카드와 나란히 두면 학생이 "내 습관 + 이번 오답의 진짜 원인"을 동시에 본다.

**필요 데이터:** 오답 원인 taxonomy(원본 문서 13.산출물 3 참조). 박승훈 입장에서 먼저 결정해야 할 것은 "taxonomy를 콘텐츠 파트너에게서 받을 것인가, 내부 fallback 버전을 코드에 하드코딩하고 시작할 것인가" 하나뿐이다.

### 기능 2. 구조화 해설 블록

**무엇:** 한 덩어리 해설이 아니라 학생이 펼쳐 보는 블록들. `정답과 핵심 근거 / 풀이 전 봐야 할 단서 / 단계별 풀이 / 선지별 판단 / 자주 하는 착각 / 같은 유형 풀이 전략 / 3분 복습 요약`.

**풀림Q에서의 위치:** `/q/analysis/[questionId]` 가운데 영역. 아코디언/탭 형태. 기본은 모두 접힌 상태, 학생 레벨(예상 등급)에 따라 기본 펼침 블록이 달라진다.

- 상위권(1~2등급): "정답 근거"와 "같은 유형 전략"만 펼침.
- 중위권(3~4등급): "단계별 풀이"와 "선지별 판단"까지 펼침.
- 하위권(5~9등급): "배경지식"과 "용어"까지 펼침.

**현재 페이지와의 연결:** 강점 분석 블록의 "예상 등급" 데이터를 그대로 재사용해서 깊이를 자동 조절할 수 있다. **즉, 풀림Q는 이미 학생 레벨을 알고 있으니, FaaS의 "수준별 depth"가 별도 설문 없이 즉시 동작한다.** 이게 다른 학습 서비스 대비 풀림Q의 큰 무기다.

### 기능 3. 개념·배경지식·용어 사전 패널

**무엇:** 문제 안에 숨은 학습 재료(어휘, 동의어/반의어, 문장 구조, 개념, 배경지식, 자료 해석법)를 옆에서 바로 본다.

**풀림Q에서의 위치:** `/q/analysis/[questionId]` 우측 사이드 패널. 데스크탑은 고정, 모바일은 하단 시트.

**현재 페이지와의 관계:** 풀림Q "코치" 메뉴와 충돌할 수 있어 보이지만 역할이 다르다. 코치는 **자유 대화형**이고, 이 패널은 **문제 컨텍스트에 고정된 정적 학습 재료**다. 패널 아이템에 "코치에게 더 묻기" 버튼을 두면 두 기능이 자연스럽게 이어진다.

**과목별 채워야 할 슬롯은 원본 정의서 8.기능 3 참조.** MVP는 국어/영어만 채우면 된다.

### 기능 4. 유형 분류와 변형 문제 생성

**무엇:** 문제를 특정 유형으로 태깅하고, 같은 구조의 유사/변형 문제로 이어준다. 액션: 같은 유형 다시 풀기 / 같은 지문 다른 문제 / 난이도 ± / 유사 기출.

**풀림Q에서의 위치:** `/q/analysis/[questionId]` 하단. **"무한풀기" 메뉴와 직접 연결되는 지점**이다. 변형 문제 카드를 누르면 무한풀기 세션으로 들어가도록 라우팅한다.

**현재 페이지와의 시너지:** 풀림Q는 이미 "무한풀기"라는 강력한 출구를 갖고 있다. 변형 문제 생성은 무한풀기의 **입력 소스**가 된다. 즉, FaaS가 풀림Q 무한풀기의 콘텐츠 엔진을 보강하는 구조가 된다.

### 기능 5. 자동 오답노트와 다음 학습 경로

**무엇:** 문제 풀이 후 오답노트 자동 생성. 다음 학습 카드(같은 유형 쉬운 문제 2개, 놓친 개념 카드 1개, 유사 기출 1개, 내일 다시 풀 문제 1개, 배경지식 카드 1개).

**풀림Q에서의 위치:** `/q/analysis/[questionId]` 페이지 맨 아래 "다음 학습" 섹션. 그리고 풀림Q **"복습" 메뉴(망각곡선 기반)에 자동 push**.

**현재 페이지와의 관계:** 복습 탭이 망각곡선 기반이라면 이미 스케줄링 엔진이 있는 것이다. FaaS는 그 큐에 **풍부한 메타데이터를 가진 항목**(왜 틀렸는지, 놓친 개념, 다음 추천)을 공급하는 역할을 한다. 즉, 복습 탭 자체를 바꿀 필요 없이 데이터만 더 부어주면 된다.

## 5. 화면 통합 제안 — 어디에 끼워 넣을 것인가

### 5-1. `/q/analysis` (현재 페이지) 변경

기존 두 블록(자기 점검 / 강점 분석)은 그대로 둔다. 다음을 추가한다.

```text
[상단 메타: 마지막 진단 2일 전 / 단원별 실력 + 풀이 습관]
   ↓
[NEW] 최근 오답 원인 Top 3 (한 줄 미니카드)
   ↓
[기존] 자기 점검 (54점, 풀이 습관 자세히)
   ↓
[기존] 강점 분석 (수학 3등급, 단원별 자세히)
   ↓
[NEW] 다시 봐야 할 문제 (최근 오답 카드 3~5개)
       └ 카드 클릭 → /q/analysis/[questionId]
   ↓
[NEW] 오늘의 복습 경로 (복습 탭 미리보기 2~3개)
```

원칙: 거시 진단을 위에, 미시 행동을 아래에. 학생은 위에서 "내가 누구인지" 확인하고, 아래에서 "지금 무엇을 할지" 정한다.

### 5-2. `/q/analysis/[questionId]` (신규 페이지) 레이아웃

```text
[상단]
- 문제 제목 / 과목·단원·유형 / 난이도
- 정답 여부 (정답/오답)
- 오답 원인 hero 카드 (기능 1)

[좌측 - 원본]
- 문제 이미지 또는 텍스트
- 학생이 고른 답 / 정답
- 지문 하이라이트 (근거 문장 표시)

[가운데 - 해설]
- 구조화 해설 블록 (기능 2, 학생 등급에 따라 자동 펼침)

[우측 - 학습 재료]
- 개념 / 용어 / 배경지식 / 동의어·반의어 (기능 3)
- "코치에게 더 묻기" 버튼

[하단]
- 유형 분류 + 변형 문제 카드 (기능 4)
   └ "이 유형으로 무한풀기" CTA
- 자동 오답노트 미리보기 + 복습 큐 등록 상태 (기능 5)
- 다음 학습 카드 5개
```

모바일에서는 좌/가운데/우를 세로 스택, 우측 패널은 하단 시트로 전환.

### 5-3. 기존 네비게이션과의 관계

- **무한풀기** ← 기능 4의 변형 문제 카드가 입력으로 흘러들어감.
- **코치** ← 기능 3 패널의 "더 묻기" 버튼이 컨텍스트와 함께 진입.
- **분석** ← 본 문서의 모든 신규 블록이 여기에 추가.
- **복습** ← 기능 5의 오답노트가 자동 push.

즉, 풀림Q의 4개 메뉴를 다 바꾸지 않고도 한 페이지(`/q/analysis/[questionId]`)와 분석 페이지 하단 추가만으로 흐름이 닫힌다.

2. `explanationBlocks`는 배열이라 새 블록 타입 추가가 비파괴적. 콘텐츠 파트너가 블록을 늘리는 동안 프런트는 안 건드려도 된다.

## 7. 단계적 도입 로드맵 (풀림Q 입장)

원본 정의서의 MVP는 자체 트랙으로 진행되더라도, 풀림Q에 통합되는 입장에서는 다음 단계가 현실적이다.

**Phase 0 — 1주차: 분석 페이지 하단만**
- `/q/analysis`에 "다시 봐야 할 문제" 카드 리스트만 추가.
- 카드 클릭 시 일단 무한풀기 결과창으로 라우팅(임시).
- 데이터는 풀림Q 기존 오답 기록 + 오답 원인 1줄 텍스트(임시 fallback)만 사용.
- **이 단계의 의미: 학생이 "문제 단위 진입점"을 실제로 누르는지 클릭률만 측정.**

**Phase 1 — 2~3주차: 문제 상세 페이지 골격**
- `/q/analysis/[questionId]` 라우트 신설.
- 기능 1(오답 원인 카드)과 기능 2(구조화 해설 블록 3개만: 정답 근거 / 단계별 풀이 / 선지별 판단) 우선 구현.
- 콘텐츠는 FaaS 트랙에서 받는 샘플 문제 5~10개로 시작.

**Phase 2 — 4~5주차: 학습 재료와 다음 학습**
- 기능 3(개념·용어 패널) + 기능 5(오답노트 자동 생성, 복습 탭 push).
- 복습 탭과의 데이터 연동 확정.

**Phase 3 — 6주차 이후: 변형 문제와 무한풀기 연결**
- 기능 4(유형 분류, 변형 문제) 추가.
- "이 유형으로 무한풀기" 라우팅으로 무한풀기 콘텐츠 엔진과 연결.

이 순서로 가면 매 단계마다 학생에게 보이는 가치가 1단계씩 추가되고, 풀림Q의 기존 메뉴(무한풀기, 코치, 복습)를 마지막에 연결한다.

## 8. 풀림Q 입장에서 미리 결정해야 할 것

콘텐츠 파트너(FaaS 트랙) 쪽이 아니라, **승훈 씨가 개발 의사결정자로서 먼저 정해야 할 것**만 추렸다.

1. `/q/analysis/[questionId]`를 새 라우트로 만들 것인지, 기존 무한풀기 결과창을 확장할 것인지.
2. 학생 등급 정보(`predictedGrade`)를 분석 페이지의 데이터 소스 그대로 재사용할 수 있는지 — 그렇다면 수준별 depth가 무료로 따라온다.
3. 복습 탭의 망각곡선 큐에 외부에서 항목을 push하는 API가 이미 있는지, 새로 만들어야 하는지.
4. 오답 원인 taxonomy가 콘텐츠 파트너에서 늦게 올 가능성을 대비해, 풀림Q 측에 fallback 4~5개(예: "근거 누락 / 어휘 오해 / 조건 누락 / 계산 실수 / 유형 전략 부재")만이라도 하드코딩으로 깔아둘지.
5. 변형 문제 생성을 LLM 실시간 호출로 갈지, 사전 생성 후 캐시로 갈지 — 무한풀기 응답속도와 연관된다.

이 5개만 정해지면 Phase 0~1은 콘텐츠가 적게 와도 골격을 먼저 깔 수 있다.

## 10. 한 줄 요약

> 풀림Q `/q/analysis`는 "학습자 거시 진단"을 잘 하고 있다. 여기에 "문제 한 개를 여는 미시 학습 허브"(`/q/analysis/[questionId]`)를 한 페이지 더 깔고, 그 출구를 무한풀기/코치/복습으로 연결하면, 풀림Q는 분석에서 끝나지 않고 학습 행동까지 닫히는 제품이 된다.
