# Pullim 문제 기반 학습 대시보드 FaaS 프로젝트 정의서

작성일: 2026-05-12  
프로젝트명: Pullim 문제 기반 학습 대시보드 FaaS  
담당 제안: 최선혜/선애 씨  
제안자: 김의연  
문서 목적: 기존 참고서·문제집 해설의 한계를 깨는 새로운 학습 경험을 FaaS 형태로 정의하고, 담당자가 바로 기획·검증을 시작할 수 있도록 배경, 핵심 기능, 세부 요소, 산출물, 완료 기준을 명확히 정리한다.

## 1. 프로젝트 한 줄 정의

학생이 문제를 풀고 난 뒤 단순 정답 해설을 보는 것이 아니라, 문제 하나를 중심으로 정답 근거, 풀이 전략, 오답 원인, 배경지식, 용어 사전, 유사·변형 문제, 오답노트, 다음 학습 경로까지 연결해주는 구조화 학습 대시보드 FaaS를 만든다.

## 2. 프로젝트 배경

대표가 교보문고 등 서점에서 여러 출판사의 자습서, 참고서, 문제집을 살펴보며 발견한 핵심 문제는 다음과 같다.

현재 시중 문제집의 해설은 출판사와 교재가 달라도 대체로 비슷한 구조를 가진다.

```text
정답 제시
간단한 근거 설명
선지별 간단 해설
일부 개념 요약
```

이 구조는 상위권 학생에게는 충분할 수 있다. 상위권 학생은 짧은 해설만 보고도 본인이 놓친 지점, 풀이 전략, 관련 개념, 다음 학습 방향을 스스로 복원할 수 있기 때문이다.

하지만 대부분의 학생은 그렇지 않다. 많은 학생은 해설을 읽어도 다음 질문을 해결하지 못한다.

```text
내가 왜 틀렸는가?
문제를 풀 때 어떤 단서를 먼저 봤어야 하는가?
이 문제는 어떤 유형인가?
내가 놓친 교과 개념은 무엇인가?
지문이나 선지의 어떤 표현이 함정이었는가?
다음에 비슷한 문제를 만나면 어떤 순서로 풀어야 하는가?
이 문제와 연결되는 배경지식은 무엇인가?
내가 다시 풀어볼 문제는 무엇인가?
```

기존 해설은 정답을 설명하지만, 학생의 사고 과정을 교정하거나 다음 학습으로 이어주는 데 약하다.

Pullim은 이 기준을 깨야 한다. 단순히 해설 문장을 더 길게 쓰는 것이 아니라, 문제 풀이 이후의 학습 경험 전체를 기능화해야 한다.

## 3. 문제 정의

기존 참고서 해설의 핵심 한계는 다음과 같다.

1. 정답 중심이다.
2. 학생의 오답 원인을 진단하지 않는다.
3. 풀이 전략을 명시적으로 가르치지 않는다.
4. 배경지식, 교과 개념, 어휘, 문장 구조가 분리되어 있다.
5. 학생 수준별로 다른 깊이의 해설을 제공하지 않는다.
6. 같은 문제를 다른 유형으로 확장하거나 변형하지 않는다.
7. 오답노트와 복습 경로로 자연스럽게 이어지지 않는다.
8. 학생이 “다음에 어떻게 풀어야 하는지”를 학습하지 못한다.
9. 교사나 학부모가 학생의 약점을 구조적으로 파악하기 어렵다.
10. AI 시대에도 여전히 종이 참고서식 해설 구조를 그대로 따르고 있다.

## 4. 제품 가설

Pullim의 차별점은 “더 친절한 해설”이 아니라 “문제 기반 학습 허브”다.

제품 가설:

```text
학생이 문제 또는 문제집 캡처를 업로드하면,
AI가 문제를 분석하고,
학습자가 놓친 사고 과정과 개념을 구조화된 콘텐츠 블록으로 제공하며,
오답노트와 다음 학습 경로까지 자동으로 연결하면,
기존 참고서 해설보다 학습 지속성과 이해도가 높아진다.
```

이 프로젝트는 이 가설을 작고 빠르게 검증하기 위한 FaaS다.

## 5. FaaS 정의

FaaS는 Feature as a Service를 의미한다.

이 프로젝트의 FaaS는 Pullim 전체 제품 안에 붙일 수 있는 독립 기능 단위다.

입력:

- 학생이 푼 문제 이미지.
- 문제집 캡처.
- 텍스트로 입력된 문제.
- 자체 제작 샘플 문제.
- 공개 또는 사용 허가된 문제.

처리:

- 문제 구조 분석.
- 정답 및 근거 분석.
- 오답 원인 분류.
- 관련 개념/용어/배경지식 추출.
- 과목별 학습 블록 생성.
- 유사·변형 문제 추천 또는 생성.
- 오답노트 자동 생성.
- 다음 학습 경로 추천.

출력:

- 구조화된 학습 대시보드.
- 학생용 해설 블록.
- 교사용 진단 요약.
- 오답노트.
- 다음 학습 카드.

## 6. 대상 사용자

### 1차 사용자: 학생

특히 다음 학생을 주요 대상으로 본다.

- 해설을 읽어도 왜 틀렸는지 모르는 학생.
- 개념은 아는 것 같은데 문제 적용이 약한 학생.
- 지문이나 선지의 함정 표현에 자주 걸리는 학생.
- 오답노트를 꾸준히 만들기 어려운 학생.
- 비슷한 유형을 반복해서 연습해야 하는 학생.

### 2차 사용자: 교사

교사는 다음 용도로 쓸 수 있다.

- 학생별 오답 원인 확인.
- 수업 보충 자료 생성.
- 유사 문제 추천.
- 개념별 취약점 파악.
- 과제 이후 피드백 자동화.

### 3차 사용자: 학부모/관리자

학부모와 관리자는 다음을 확인할 수 있다.

- 아이가 무엇을 틀렸는지보다 왜 틀렸는지.
- 다음에 무엇을 공부해야 하는지.
- 단순 점수보다 학습 경로가 개선되고 있는지.

## 7. 담당자에게 기대하는 역할

최선혜/선애 씨에게 기대하는 역할은 단순 문서 정리자가 아니다.

기대 역할:

```text
Education Content Quality Lead
Learning Design Lead
Explanation 2.0 Architect
```

담당자가 해야 할 핵심 판단:

1. 좋은 해설과 나쁜 해설의 기준 정의.
2. 과목별로 필요한 학습 블록 정의.
3. 학생 수준별 해설 깊이 설계.
4. 오답 원인 taxonomy 설계.
5. AI가 생성한 콘텐츠를 사람이 어떻게 검수할지 기준화.
6. 기존 참고서 해설과 Pullim식 해설의 차이를 샘플로 증명.
7. 교사와 학생이 실제로 쓸 수 있는 화면 콘텐츠 구조 제안.

담당자에게 맡기면 안 되는 일:

- 단순 프로젝트 목록 정리.
- 기존 자료 표 정리만 하는 일.
- AI가 만든 내용을 검토 없이 옮기는 일.
- 기능 이름만 나열하는 일.

## 8. 핵심 기능 5가지

MVP 대시보드에는 아래 5개 기능을 우선 포함한다.

### 기능 1. 오답 원인 진단 카드

학생이 틀린 이유를 유형화해 보여주는 기능이다.

예시 오답 원인:

```text
지문 근거를 놓침
단어 의미를 오해함
선지의 범위를 과대 해석함
문제 조건을 끝까지 읽지 않음
개념 A와 개념 B를 혼동함
계산 실수
자료 해석 순서 오류
문제 유형의 풀이 전략을 모름
배경지식 부족
문장 구조 해석 실패
```

화면에서 보여줄 내용:

- 학생이 고른 답.
- 정답.
- 예상 오답 원인.
- 근거 문장 또는 조건.
- 다음에 같은 실수를 줄이는 방법.

담당자 산출물:

- 오답 원인 taxonomy v1.
- 과목별 오답 원인 예시.
- 오답 원인별 피드백 문장 샘플.

### 기능 2. 구조화 해설 블록

기존처럼 하나의 긴 해설 문단을 제공하지 않고, 학생이 필요한 부분을 선택적으로 볼 수 있는 블록형 해설을 제공한다.

기본 블록:

```text
정답과 핵심 근거
문제를 풀기 전 봐야 할 단서
단계별 풀이 과정
선지별 판단
자주 하는 착각
다음에 같은 유형을 푸는 전략
3분 복습 요약
```

설계 방향:

- 상위권 학생은 정답 근거와 전략만 빠르게 본다.
- 중위권 학생은 단계별 풀이와 선지별 판단을 본다.
- 하위권 학생은 배경지식, 용어, 기초 개념까지 함께 본다.

담당자 산출물:

- 해설 블록 구조 v1.
- 학생 수준별 depth 3단계.
- 좋은 해설/나쁜 해설 예시.
- 샘플 문제 5개 before/after.

### 기능 3. 개념·배경지식·용어 사전 패널

문제 안에 숨어 있는 학습 재료를 추출해, 해설 옆에서 바로 볼 수 있게 한다.

과목별 예시:

국어:

- 핵심 어휘.
- 지문 배경.
- 문단 구조.
- 표현 방식.
- 선지 함정 표현.
- 문학 작품/작가/시대 배경.

영어:

- 단어 뜻.
- 동의어.
- 반의어.
- 문장 구조.
- 구문 해석.
- paraphrase.
- 지문 요지.
- 빈칸/순서/삽입 문제 전략.

수학:

- 사용 개념.
- 필요한 공식.
- 조건 해석.
- 풀이 순서.
- 자주 하는 계산 실수.
- 같은 개념의 변형 유형.

사회/과학:

- 핵심 개념.
- 배경지식.
- 자료 해석 방법.
- 그래프/표 읽는 순서.
- 관련 교과 단원.
- 오개념 주의.

담당자 산출물:

- 과목별 필수 패널 정의.
- 용어 사전 entry 구조.
- 영어 동의어/반의어 카드 구조.
- 교과 개념 연결 규칙.

### 기능 4. 유형 분류와 변형 문제 생성

문제를 특정 유형으로 분류하고, 같은 구조의 유사 문제 또는 변형 문제로 이어준다.

예시:

```text
국어: 세부 정보 추론형, 문단 구조 파악형, 표현 방식 판단형
영어: 빈칸 추론형, 순서 배열형, 문장 삽입형, 요지 파악형
수학: 조건 해석형, 개념 적용형, 함수 그래프 해석형
과학: 자료 해석형, 실험 설계형, 개념 적용형
사회: 사례 적용형, 개념 비교형, 자료 분석형
```

화면 액션:

- 같은 유형으로 다시 풀기.
- 같은 지문으로 다른 문제 만들기.
- 난이도 낮춰서 다시 풀기.
- 난이도 올려서 도전하기.
- 유사 기출 보기.
- 문제 구조를 바꿔서 풀기.

담당자 산출물:

- 과목별 문제 유형 taxonomy v1.
- 변형 문제 생성 규칙.
- 난이도 조절 기준.
- 유사 문제 추천 기준.

### 기능 5. 자동 오답노트와 다음 학습 경로

문제를 푼 뒤 자동으로 오답노트와 다음 학습 경로가 생성된다.

오답노트 구성:

```text
문제 요약
내가 고른 답
정답
틀린 이유
놓친 개념
다시 봐야 할 단어/용어
같은 유형 풀이 전략
유사 문제
복습 예정일
다음 추천 학습
```

다음 학습 추천 예:

```text
같은 유형 쉬운 문제 2개
놓친 개념 카드 1개
유사 기출 1개
내일 다시 풀 문제 1개
관련 배경지식 카드 1개
```

담당자 산출물:

- 오답노트 템플릿 v1.
- 다음 학습 추천 규칙.
- 복습 카드 구조.
- 교사용 요약 구조.

## 9. 대시보드 화면 구성 제안

### 기본 레이아웃

```text
상단:
- 문제 제목
- 과목/단원/유형
- 정답 여부
- 난이도
- 주요 오답 원인

왼쪽:
- 원본 문제 이미지 또는 텍스트
- 내가 고른 답
- 정답
- 지문/문제 하이라이트

가운데:
- 구조화 해설 블록
- 풀이 순서
- 선지별 판단
- 같은 유형 풀이 전략

오른쪽:
- 개념 패널
- 용어 사전
- 배경지식
- 영어 동의어/반의어
- 관련 교과 단원

하단:
- 유사 문제
- 변형 문제
- 오답노트
- 다음 학습 경로
```

### 화면에서 중요한 UX 원칙

1. 한 번에 모든 정보를 펼치지 않는다.
2. 학생 수준에 따라 해설 깊이를 조절한다.
3. 정답보다 “왜 그렇게 생각해야 하는지”를 먼저 보여준다.
4. 오답을 부끄럽게 만들지 않고, 학습 경로로 전환한다.
5. 해설, 개념, 어휘, 유사 문제가 한 화면에서 연결되어야 한다.

## 10. 콘텐츠 블록 전체 후보

MVP에서는 5개 핵심 기능만 우선 구현하지만, 장기적으로 다음 콘텐츠 블록을 확장할 수 있다.

| 블록 | 설명 | 우선순위 |
|---|---|---|
| 정답 근거 | 왜 이 답이 맞는지 핵심 근거 제시 | P0 |
| 풀이 순서 | 어떤 순서로 접근해야 하는지 | P0 |
| 오답 원인 | 학생이 왜 틀렸는지 유형화 | P0 |
| 선지별 판단 | 각 선택지를 어떻게 판단해야 하는지 | P0 |
| 개념 연결 | 관련 교과 개념 연결 | P0 |
| 용어 사전 | 문제 속 용어/단어 설명 | P0 |
| 배경지식 | 지문/개념 이해에 필요한 배경 | P1 |
| 동의어/반의어 | 영어 어휘 확장 | P1 |
| 문장 구조 | 영어/국어 문장 구조 분석 | P1 |
| 유형 분류 | 문제 유형 태깅 | P0 |
| 유사 문제 | 같은 유형의 문제 추천 | P1 |
| 변형 문제 | 같은 구조를 다른 문제로 변환 | P1 |
| 난이도 조절 | 쉬운 버전/어려운 버전 생성 | P2 |
| 오답노트 | 자동 기록 및 복습 카드 | P0 |
| 다음 학습 | 복습/개념/문제 추천 | P0 |
| 교사용 요약 | 학생 취약점 요약 | P2 |

## 11. MVP 범위

1차 MVP는 제품 전체를 만드는 것이 아니라, 이 기능이 실제로 기존 해설보다 나은지 검증하는 것이다.

### MVP 입력

- 자체 제작 또는 사용 가능한 샘플 문제 20개.
- 과목은 우선 2개만 선택한다.
- 권장: 국어 10개, 영어 10개.
- 이유: 최선혜/선애 씨의 국문과, 교육대학원, EBS 경력과 가장 잘 맞고, 용어/해석/배경지식/문장 구조의 차별점이 잘 드러난다.

### MVP 출력

각 문제당 다음을 만든다.

```text
1. 기존 참고서식 해설
2. Pullim식 구조화 학습 대시보드 콘텐츠
3. 오답 원인 카드
4. 개념/용어/배경지식 패널
5. 유사 또는 변형 문제 제안
6. 자동 오답노트
7. 다음 학습 경로
```

### MVP에서 하지 않을 것

- 완전한 OCR 시스템 구축.
- 모든 과목 대응.
- 시중 문제집 원문 대량 저장.
- 출판사 저작권이 걸린 문제의 무단 DB화.
- 완성된 LMS 구축.
- 학생 계정/학습 이력 전체 시스템 구축.

## 12. 저작권과 데이터 사용 원칙

문제집 캡처 업로드 기능은 저작권 이슈가 있을 수 있다.

따라서 MVP에서는 아래 원칙을 따른다.

1. 자체 제작 문제 또는 공개/허가된 문제를 우선 사용한다.
2. 시중 문제집 캡처는 내부 검토 샘플로만 제한하고, 저장/재배포하지 않는다.
3. 사용자가 올린 이미지는 분석 목적의 임시 처리로 설계한다.
4. 원문 문제를 서비스 콘텐츠로 축적하는 방식은 피한다.
5. 출판사 제휴 가능성을 별도 사업 모델로 검토한다.
6. 화면/문서에는 저작권 출처와 사용 범위를 명확히 표시한다.

## 21. 최종 요약

이 프로젝트는 해설 문장을 조금 더 친절하게 바꾸는 일이 아니다.

Pullim이 해야 할 일은 문제 하나를 정답 확인 단위가 아니라, 개념, 어휘, 배경지식, 사고 전략, 오답 원인, 변형 문제, 복습 경로가 열리는 학습 허브로 바꾸는 것이다.

최선혜/선애 씨는 이 프로젝트에서 기존 참고서 해설의 한계를 깨는 교육 콘텐츠 기준을 만들고, AI가 생성한 결과물을 사람이 검수할 수 있는 품질 체계를 설계해야 한다.
