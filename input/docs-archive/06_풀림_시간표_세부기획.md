## 2. 왜 이 기능이 필요한가

### 2.1 시장 문제

| 문제 | 현상 | 규모 |
|------|------|------|
| 계획만 세우고 실행 안 됨 | 학생의 70%가 시험 공부 계획을 세우지만, 50% 이상이 첫째 날 이탈 | 고등학생 설문 기반 |
| 과목 배분 감 없음 | "수학 많이 해야지" → 실제로는 좋아하는 과목만 함 | 학습 편향 보편적 |
| 계획-실행 연결 끊김 | 노션에 "수학 2시간" 적어도 실제로 뭘 풀지 모름 | 범용 도구의 한계 |
| 리밸런싱 불가 | 어제 못한 걸 내일 어떻게 보충할지 스스로 판단 어려움 | 자기조절학습 미숙 |

### 2.2 풀림 시간표의 답

핵심 차별화: **"시간표에 적힌 것이 '수학 30분'이 아니라, '수학I 삼각함수 사인법칙 — 난이도 0.5~0.7 문제 8개 + 어제 틀린 패턴 복습 2개'"**

이것은 다음이 결합되어야만 가능:
- **3-Depth 교육과정 분류**: 어떤 단원의 어떤 세부 유형을 공부할지 결정
- **IRT 파라미터 (θ)**: 현재 실력 대비 최적 난이도 범위 산출
- **오답 패턴 DB**: 어떤 유형에서 반복 실수하는지 반영
- **스페이스드 리피티션**: 언제 복습해야 기억이 유지되는지 계산

### 2.3 경쟁사 비교

| | Notion AI / 구글 캘린더 | 뤼이드 산타 | ChatGPT | **풀림 시간표** |
|---|---|---|---|---|
| 계획 생성 | 범용 일정 | 단원 순서만 | 텍스트 플랜 | **분 단위 시간 블록** |
| 실행 연동 | 없음 | 앱 내 문제풀이 | 없음 | **블록 탭 → 즉시 실행** |
| 콘텐츠 배치 | 없음 | 고정 문제셋 | 없음 | **IRT 기반 자동 배치** |
| 취약점 반영 | 없음 | 기본 순서 변경 | 없음 | **인덱스 θ → 시간 비율 산출** |
| 적응형 조정 | 없음 | 순서 변경 | 없음 | **매일 잔여 플랜 재최적화** |
| 학습 자료 종류 | 없음 | 문제만 | 없음 | **7가지 블록 유형** |

## 3. 서비스 대상

### 3.1 주요 사용자

| 사용자 | 사용 시나리오 | 핵심 니즈 |
|--------|-----------|----------|
| 고등학생 (자기주도) | 시험 대비 시간표 자동 생성·실행 | "뭘 얼마나 해야 하는지" 알려줘 |
| 학부모 | 자녀 학습 시간표 생성·관리 | 자녀 학습량 모니터링 |
| 과외선생/강사 | 학생별 맞춤 학습 계획 설계 | 수업 외 자습 시간 관리 |
| 재수생 | 장기 수능 대비 플랜 | D-day 역산 최적 배분 |
| (향후) 자격증 수험생 | 시험 대비 플랜 | 과목별 시간 배분 최적화 |

### 3.2 역할별 권한

| 역할 | 시간표 생성 | 시간표 수정 | 실행 | 리포트 조회 |
|------|-----------|-----------|------|-----------|
| 학습자 (learner) | O (자기 것) | O (자기 것) | O | 본인만 |
| 운영자 (manager) | O (학생 것도) | O (학생 것도) | - | 관리 학생 전체 |
| 보조자 (assistant) | - | 승인 요청 → manager에게 알림 → 48시간 내 미응답 시 자동 승인 | - | 관리 학생 전체 (열람) |

## 4. 모듈 구조

### 4.1 전체 아키텍처

```
planner/
├── core/
│   ├── plan-engine/           # 플랜 생성·재생성·조정 핵심 엔진
│   ├── time-block-manager/    # 시간 블록 CRUD + 상태 관리
│   ├── execution-tracker/     # 실행 추적 + 완료 체크
│   └── adaptive-rebalancer/   # 매일 재최적화 엔진
│
├── intelligence/
│   ├── diagnostic-bridge/     # 인덱스 슬롯에서 θ 조회 + 미니 진단
│   ├── cognitive-load-model/  # 인지 부하 기반 시간대 배치
│   ├── constraint-solver/     # 제약조건 만족 스케줄링 (CSP)
│   └── goal-distance-calc/    # 목표 달성 거리 산출기
│
├── content-orchestrator/      # 시간 블록별 콘텐츠 자동 배치
│   ├── block-type-router/     # 블록 유형 → 슬롯 매핑
│   ├── material-fetcher/      # 슬롯에서 실제 콘텐츠 가져오기
│   └── fallback-generator/    # 슬롯 미연결 시 자체 콘텐츠 생성
│
├── ui/
│   ├── input-wizard/          # 시험 정보 입력 위저드
│   ├── timeline-view/         # 일별·주별 타임라인 뷰
│   ├── block-executor/        # 블록 실행 화면 (풀 임베디드)
│   └── progress-dashboard/    # 달성률·성장 대시보드
│
├── adapters/
│   ├── curriculum-adapter/    # 도메인별 교육과정 (수업방과 공유)
│   └── calendar-adapter/      # 외부 캘린더 연동 (향후)
│
└── slots/                     # 플러그인 슬롯
    ├── infinity-slot/         # 문제풀이 블록용
    ├── tutor-slot/            # 개념 질문 즉시 연결
    ├── conqueror-slot/        # 취약점 보강 블록용
    ├── memory-slot/           # 암기 블록용
    ├── visual-slot/           # 개념학습 블록용
    └── exam-slot/             # 모의고사 블록용
```

### 4.2 플러그인 슬롯 시스템

| 슬롯 | connected 동작 | fallback 동작 |
|------|---------------|--------------|
| infinity-slot | IRT 기반 범위·난이도 자동 문제 출제 | T3로 범위 내 문제 생성 |
| tutor-slot | 소크라테스 5단계 튜터 세션 연결 | T2로 교육과정 기반 Q&A |
| conqueror-slot | 오답 패턴 기반 정복 세트 로드 | 최근 오답 재출제 (T2) |
| memory-slot | 스페이스드 리피티션 세션 실행 | 플래시카드 자체 생성 (T2) |
| visual-slot | VLM 인터랙티브 모듈 실행 | 개념 카드 + 텍스트 설명 (T2) |

## 5. 시간 블록(TimeBlock) 체계

### 5.1 7가지 블록 유형

| 블록 유형 | 아이콘 | 연결 슬롯 | 기본 시간 | 설명 |
|----------|--------|----------|----------|------|
| **개념학습** | 📖 | visual-slot | 20~30분 | 새 개념 학습, 교과서 정리 |
| **문제풀이** | ✏️ | infinity-slot | 30~45분 | IRT 기반 난이도 맞춤 문제 |
| **취약보강** | 🎯 | conqueror-slot | 20~30분 | 오답 패턴 집중 훈련 |
| **암기** | 🧠 | memory-slot | 15~20분 | 스페이스드 리피티션 |
| **모의고사** | 📝 | exam-slot | 40~90분 | 범위 지정 실전 시뮬레이션 |
| **질문타임** | 💬 | tutor-slot | 10~15분 | 개념 질문, 풀이 막힘 해결 |
| **휴식** | ☕ | 없음 | 5~15분 | 타이머만 |

### 5.2 블록 상태 머신

```
[예정] → (시작 시간 도달) → [진행중] → (완료) → [완료]
                                      → (스킵) → [스킵]
                                      → (중단) → [중단]
                              → (시간 초과) → [초과완료] or [미완료]
```

### 5.3 블록 실행 화면 (풀 임베디드)

블록을 탭하면 풀림 스터디 앱 안에서 즉시 해당 학습이 시작됨:

```
┌─────────────────────────────────────────┐
│  ✏️ 수학I 삼각함수 문제풀이  ⏱ 32:15    │
│  ─────────────────────────────────────  │
│                                         │
│  [문제 3/8]                             │
│  삼각형 ABC에서 a=4, b=6, C=60° 일 때   │
│  삼각형의 넓이를 구하시오.               │
│                                         │
│  ┌─────────────────────────────┐        │
│  │  풀이 입력 영역              │        │
│  └─────────────────────────────┘        │
│                                         │
│  [힌트 요청 💬]  [다음 문제 →]  [블록 종료]│
│                                         │
│  진행률: ███████░░░ 3/8 (37%)           │
└─────────────────────────────────────────┘
```

- "힌트 요청" → tutor-slot 즉시 연결
- 문제 완료 시 → 정답 여부 + 간단 해설 → 다음 문제
- 블록 종료 시 → 블록 결과 요약 + 다음 블록 안내

## 6. 플랜 생성 엔진

### 6.1 Step 1: 입력 위저드

```
[시험 정보]
├─ 시험 유형: 내신 중간 / 내신 기말 / 모의고사 / 수능 / 자격증 / 커스텀
├─ 시험 일자: 날짜 선택
├─ 과목 선택: 교육과정 3-Depth 트리에서 체크
│   ├─ 자연어 입력 지원: "고1 수학 2단원" → 자동 매핑
│   └─ 학교별 범위 템플릿 공유 (향후)
├─ 하루 가용 시간: 평일 / 주말 각각 설정
├─ 선호 시간대: 시간 슬라이더
└─ 컨디션 설정 (선택):
    ├─ 집중력 높은 시간: 오전 / 오후 / 저녁
    └─ 피곤한 요일: 체크박스
```

**도메인별 위저드 적응**:
- 고등 내신: 학교 선택 → 시험 범위 자동 추천 (학교별 진도 DB, 향후)
- 수능: 전 과목 범위 고정, D-day 역산
- 자격증: 과목별 출제 비율 기반 시간 가중치

### 6.2 Step 2: 진단 브릿지

```
인덱스 슬롯 조회 (connected인 과목)
  → 수학I θ = 1.2 (상위 25%)
  → 영어 θ = 0.3 (중위 55%)

인덱스 슬롯 미연결 과목 (fallback)
  → 국어: 미니 진단 10문항 퀵 테스트 (5분)
  → T2로 교육과정 기반 진단 문항 생성 → 응답 수집 → θ 초기값 추정

목표 달성 거리 산출:
  과목별 (현재 θ → 목표 θ) 기반 예상 필요 학습 시간 계산
  → 수학I: 거리 0.3 → 8시간
  → 영어: 거리 0.7 → 18시간
  → 국어: 거리 1.0 → 25시간
```

**목표 달성 거리 공식**:
```
required_hours = base_hours × (target_θ - current_θ) × difficulty_factor
  base_hours: 과목별 기본 단위 (데이터 축적 후 개인화)
  difficulty_factor: 과목 난이도 계수 (수학 1.3, 영어 1.0, 국어 0.9 등)
```

### 6.3 Step 3: 제약조건 만족 스케줄링 (CSP)

**입력 변수:**
- 가용 시간 블록 목록 (일별, 사용자 입력 기반)
- 과목별 필요 시간 (진단 기반)
- 인지 부하 모델 파라미터
- 블록 유형별 최소/최대 시간 제약

**제약조건:**
```
C1: 어려운 과목(θ 낮음)은 집중력 높은 시간대에 배치
C2: 같은 과목 연속 최대 90분 (인지 부하 상한)
C3: 과목 전환 시 5분 전환 블록(휴식) 삽입
C4: 하루 시작은 복습 블록(기억장치), 마무리는 정리 블록
C5: 주 1회 이상 모의고사 블록 (시험 2주 전부터)
C6: 취약 과목에 전체 시간의 비율 가중 배분
    → 거리 비례: 국어(1.0): 영어(0.7): 수학(0.3) = 50%: 35%: 15%
C7: 하루 총 학습 시간 ≤ 사용자 설정 가용 시간
C8: 취약보강 블록은 해당 과목 문제풀이 블록 뒤에 배치 (오답 즉시 보강)
```

**AI Tier 배분:**
- 스케줄 생성: **T2** (구조화된 JSON 출력)
- 제약조건 검증: **T1** (룰엔진으로 C1~C8 체크)
- 실패 시 재생성: T2 재호출 (최대 3회)

### 6.4 Step 4: 콘텐츠 오케스트레이터

플랜 생성 후 각 블록에 **실제 학습 자료를 사전 배치**:

```
블록 유형 판정 → 해당 슬롯 상태 확인
  → connected: Core API 호출하여 콘텐츠 사전 로드
  → fallback: fallback-generator로 자체 콘텐츠 생성

예시:
[블록: 수학I 문제풀이 45분]
  → infinity-slot: connected
  → API: { subject: "수학I", topic: "삼각함수.사인법칙",
            difficulty: [0.4, 0.7], count: 8,
            exclude: [최근 7일 풀었던 문제 ID] }
  → 결과: 8개 문제 사전 로드

[블록: 국어 개념학습 30분]
  → visual-slot: fallback
  → T2 생성: 교육과정 노드 "국어-문학-현대시" 기반
  → 개념 카드 5장 + 핵심 정리 텍스트 자동 생성
```

## 7. 적응형 리밸런서

### 7.1 작동 방식

```
매일 23:59 자동 실행 (BullMQ 스케줄)

[입력]
├─ 오늘 계획 vs 실제 실행률 (블록별)
├─ 블록별 정답률·소요시간
├─ 업데이트된 θ 값 (IRT 재추정, connected 과목)
└─ 남은 D-day

[판단 로직] (T1: 룰엔진)
├─ 실행률 < 70%?
│   → 미완료 블록 내일로 이월 + 잔여 일정 압축
├─ 특정 과목 정답률 급상승 (+15% 이상)?
│   → 해당 과목 시간 축소, 약한 과목으로 재배분
├─ 특정 과목 정답률 급하락 (-15% 이상)?
│   → 난이도 하향 + 개념학습 블록 추가 삽입
├─ D-day ≤ 7?
│   → 시험 직전 모드: 모의고사 빈도 증가 + 취약 집중
├─ D-day ≤ 3?
│   → 총정리 모드: 취약 단원만 모은 최종 시간표
└─ 변경 없음?
    → 기존 플랜 유지

[출력]
├─ 수정된 잔여 플랜 (JSON)
├─ 학생 알림: "내일 국어 시간을 30분 늘렸어. 어제 현대시 파트에서 좀 힘들었지?"
└─ 운영자 알림: 주간 조정 요약
```

### 7.2 수동 트리거

| 트리거 | 동작 |
|--------|------|
| "오늘 컨디션 안 좋아요" / 컨디션 슬라이더 | 난이도 전체 -0.2 하향, 문제풀이→복습 일부 전환 |

## 10. AI 파이프라인

### 10.1 3-Tier 모델 라우터

| Tier | 모델 | 용도 | 호출당 비용 |
|------|------|------|-----------|
| **T1: Edge** | 룰엔진 + 임베딩 | CSP 제약 검증, 인지 부하 배치, 리밸런서 판단 | ~₩0 |
| **T2: Fast** | Claude Haiku / GPT-4o-mini | 스케줄 생성, 알림 메시지, fallback 콘텐츠 | ₩5~15 |
| **T3: Deep** | Claude Sonnet / GPT-4o | 문제 생성(CUREA DEEP), 진단, 모의고사 생성 | ₩50~200 |

### 10.2 비용 최적화

| 기법 | 적용 포인트 | 절감 효과 |
|------|-----------|---------|
| **콘텐츠 사전 로드** | 블록 시작 전 미리 문제/개념 생성 → 피크 시간 분산 | 안정성 향상 |
| **Prompt Caching** | 교육과정 시스템 프롬프트 캐싱 | 15~25% |
| **블록 결과 캐시** | 동일 범위·난이도 문제 세트 재사용 (다른 학생에게) | 20~30% |
| **Batch 리포트** | 주간/월간 리포트는 야간 Batch API | 50% 할인 |
| **T1 최대 활용** | CSP, 인지 부하, 리밸런서 모두 룰엔진 | 전체 60% 절감 |

### 10.3 예상 비용 (학생 1명 기준, 월)

| 항목 | 월 사용량 | 단가 | 월 비용 |
|------|----------|------|--------|
| T1 (리밸런서+CSP 검증) | 30회/월 | ₩0 | ₩0 |
| T2 (스케줄 생성+알림+CSP 재시도) | 30회 + CSP재시도 ~10회 + 알림 ~60회 = ~100회/월 | ₩12 | ₩1,200 |
| T2 (fallback 콘텐츠) | 100블록 × 30% fallback = 30회 | ₩15 | ₩450 |
| T3 (문제 생성, connected 슬롯) | 100블록 × 40% = 40회 | ₩100 | ₩4,000 |
| T3 Batch (리포트) | 4회/월 | ₩150 | ₩600 |
| **합계** | | | **₩6,250/월** |

> **비용 산출 주의**: CSP 제약조건 검증은 T1 룰엔진이지만, CSP 실패 시 T2로 재생성(최대 3회)하는 비용 포함. 리밸런서 알림 메시지(일간+수동 트리거)도 T2에 포함.

학생 구독료 ₩19,900~29,900 기준 → **AI 원가율 21~31%** → 마진 양호

## 18. 대한민국 교육 시장 타당성

### 18.1 시장 기회

| 요인 | 근거 |
|------|------|
| 시험 시즌 폭발적 수요 | 중간·기말·모의고사 연 6~8회 → 시즌마다 마케팅 기회 |
| 자기조절학습 미숙 | 고등학생 70%가 계획은 세우지만 50% 이상 첫날 이탈 |
| 고비용 과외 대체 | 1:1 과외 월 50~100만 원 → AI 플래너 ₩19,900 |
| 재수생 시장 | 연 20만+ 명, 장기 플랜 수요 높음 |

### 18.2 핵심 판단

> 시험 시간표는 **대한민국 모든 학생이 시험 전에 하는 행위**. "시간표를 짜는 것"이 아니라 "시간표대로 바로 공부가 시작되는 것"이 결정적 차별화. 노션이나 구글 캘린더로는 절대 대체 불가능한 **콘텐츠 연결형 플래닝**이 핵심 가치.
